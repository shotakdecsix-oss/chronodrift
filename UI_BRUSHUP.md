# 伊勢原ファンタジー UIブラッシュアップ設計書

対象: `index.html` の HUD（3D探索UI）のみ。3D描画・ゲームロジックは変更しない。
実装者向け（Sonnet想定）。モックアップ `ui-mockup.html` が見た目の正。

---

## 1. 現状の課題

1. **配色がバラバラ**: 左側のボタン群が紫/緑/橙/灰/青と全て別配色。色に意味がなく視覚ノイズ。
2. **固定px配置**: 各要素が `top:96px` 等の決め打ちで、画面サイズによって重なる。safe-area未対応。
3. **常時表示が多すぎる**: 操作ヘルプ(#info)、ステータス(#status)、GPS座標、海面スライダーが常に出ており、3D世界への没入を妨げる。
4. **情報の重複**: 住所(#addressDisplay)とGPS座標(#gpsDisplay)が別々の場所に別スタイルで存在。
5. **階層がない**: 「よく使う操作」(視点切替・ジャンプ)と「たまに使う設定」(海面・時代モード)が同列に並ぶ。

## 2. デザイン方針

- **ガラス調HUDに統一**: 全HUD要素を同一マテリアルにする。
  - `background: rgba(15,18,32,0.55)` + `backdrop-filter: blur(12px)` + `border: 1px solid rgba(255,255,255,0.14)`
  - 角丸: パネル14px / 円形ボタンは50%
- **アクセントは1色**: `--accent: #8b7bff`（ファンタジー紫を継承）。アクティブ状態・フォーカスのみに使用。将来、時代モード連動でアクセント色を差し替え可能な設計にする（CSS変数1個の差し替えで済むこと）。
- **レイアウトを4領域に整理**（決め打ちpx廃止、safe-area対応）:
  - 上中央: ロケーションカプセル
  - 左端: アイコンレール（縦並び）
  - 右下: ミニマップ＋ジャンプボタン
  - 左下: ジョイスティック
- **常時表示は最小限**: ヘルプ→初回のみ＋?ボタン、ステータス→自動消滅トースト、海面→ポップオーバー。

## 3. レイアウト仕様

```
┌────────────────────────────────────────┐
│        [📍 伊勢原市田中 ▾]        [🙈] │ ← top: max(16px, env(safe-area-inset-top))
│ ┌──┐                                   │
│ │👁│ アイコンレール                     │
│ │🗺│ (48px円形×5、gap 10px、           │
│ │📍│  左端 max(16px, safe-area-left))  │
│ │🏙│                          ┌──────┐ │
│ │🌊│                          │minimap│ │
│ └──┘                          └──────┘ │
│   (toast: 下中央、自動消滅)      [⤴]   │
│ (joystick)                             │
└────────────────────────────────────────┘
```

### 3.1 ロケーションカプセル（新規、#addressDisplay と #gpsDisplay を統合）
- 上中央。ガラス調ピル。`📍 神奈川県伊勢原市田中` を常時表示。
- タップで下に展開: 緯度経度・標高・「Googleマップで開く」リンク（既存 #gpsDisplay の href を流用）。
- 展開状態は再タップ or 3D画面タップで閉じる。

### 3.2 アイコンレール（既存5ボタン+海面を統合）
- 48px円形ガラスボタン、縦並び、gap 10px。絵文字をアイコンとして使用（サイズ20px、中央揃え）。
- 並び順（使用頻度順）: 👁視点切替 / 🗺上空 / 📍マップジャンプ / 🏙時代モード / 🌊海面
- ラベル: ボタン右に小さなツールチップ（hover時のみ。タッチでは非表示でよい）。
- トグル系（視点・上空）はアクティブ時にアクセント色のリング `box-shadow: 0 0 0 2px var(--accent)`。
- 🏙時代モードはタップごとに巡回（現行踏襲）。現在モードをボタン下に7px程度の小ラベルで表示（現実/明治/江戸/メルヘン/宇宙）。
- 🌊海面: タップでボタン右にポップオーバー（ガラス調カード、幅220px）を開き、既存スライダーと数値を格納。外側タップで閉じる。
- 🔄視点(camDirBtn)は👁視点ボタンの長押し…にはせず、レールの6個目として残してよい（実装を単純に保つ）。

### 3.3 トースト（#status 置き換え）
- 下中央（ジョイスティックと重ならないよう bottom: 96px 目安）。
- 表示3秒→フェードアウト。連続時は最新のみ。読み込み中系（「🗺 マップ読み込み中…」）は完了まで保持し完了メッセージに差し替え。
- 既存コードの `status.textContent = ...` 箇所を `showToast(msg, {sticky})` 関数に置換する。

### 3.4 ヘルプ（#info 置き換え）
- 初回起動時（localStorage フラグ）のみモーダル表示。以後はレール最下部の「?」ボタンから開く。
- 内容は現行の操作説明そのまま。

### 3.5 そのまま維持するもの
- ジョイスティック / ⤴ホップボタン / ミニマップ / 🙈UIトグル / マップジャンプのオーバーレイ（ただし配色はガラス調トークンに合わせる）。
- 🙈トグルの非表示対象リストは新要素IDに合わせて更新。

## 4. デザイントークン（:root に定義）

```css
:root {
  --hud-bg: rgba(15,18,32,0.55);
  --hud-border: rgba(255,255,255,0.14);
  --hud-blur: 12px;
  --hud-radius: 14px;
  --accent: #8b7bff;          /* 時代モード連動で将来差し替え */
  --text: #eef0fa;
  --text-dim: rgba(238,240,250,0.65);
  --shadow: 0 4px 24px rgba(0,0,0,0.35);
  --safe-top: max(16px, env(safe-area-inset-top));
  --safe-left: max(16px, env(safe-area-inset-left));
  --safe-right: max(16px, env(safe-area-inset-right));
  --safe-bottom: max(16px, env(safe-area-inset-bottom));
}
```

- フォント: `font-family: "Hiragino Sans","Yu Gothic","Noto Sans JP",sans-serif;`（他アプリと統一）。座標・標高の数値は `font-variant-numeric: tabular-nums`。
- `backdrop-filter` 非対応環境では背景の不透明度を0.8に上げるフォールバック（`@supports not (backdrop-filter: blur(1px))`）。

## 5. 実装手順（推奨順）

1. `<style>` 内をトークン定義＋新スタイルに置き換え（3Dキャンバス関連は不変）。
2. HTML: ボタン群→`<nav id="hudRail">`、住所+GPS→`<div id="locCapsule">`、#status→`<div id="toast">`、#info→ヘルプモーダルに再構成。**既存IDはできる限り温存**し、JSの `getElementById` 参照が壊れないようにする（viewBtn/mapBtn/jumpBtn/modeBtn/camDirBtn/seaSlider/seaVal/minimap/joystick/hopBtn/uiToggleBtn は同IDのまま新レイアウトへ移動）。
3. JS: `showToast()` 追加と #status 参照箇所の置換、ロケーションカプセル展開、海面ポップオーバー開閉、初回ヘルプ。ゲームロジックは触らない。
4. 動作確認: スマホ縦(375px)・スマホ横・PCで重なりがないこと。

## 6. 受け入れ基準

- [ ] 全HUD要素が同一のガラス調マテリアル・同一角丸で表示される
- [ ] 固定px縦位置の連鎖（top:56/96/136/176/216px）が排除され、flexレールになっている
- [ ] iPhoneのノッチ/ホームバーとHUDが重ならない（safe-area）
- [ ] 住所・座標・標高が1つのカプセルに統合され、タップで展開する
- [ ] #status が自動消滅トーストになっている
- [ ] 操作ヘルプは初回のみ表示、以後?ボタンから
- [ ] 3D描画・移動・ジャンプ・マップジャンプ・時代切替・海面変更の全機能が従来通り動く
